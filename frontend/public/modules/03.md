# Modulo 3: 📦 Risorse di Base

In Kubernetes tutto è una risorsa dichiarata in YAML.

## 1. Pod
L'unità più piccola che puoi creare in Kubernetes. Un Pod contiene uno o più container che condividono rete e storage.

## 2. Deployment
Invece di creare singoli Pod, usiamo i Deployment per gestire la scalabilità e gli aggiornamenti. Se un Pod fallisce, il Deployment lo ricrea.

## 3. Service
I Pod hanno IP dinamici che cambiano a ogni riavvio. Un Service fornisce un IP stabile e un nome DNS per raggiungere un gruppo di Pod.
- **ClusterIP**: Solo accesso interno.
- **NodePort**: Espone il servizio su una porta fissa in ogni nodo.
- **LoadBalancer**: Crea un bilanciatore di carico esterno (tipico dei cloud provider).

## 4. ConfigMap & Secret
Separare il codice dalla configurazione:
- **ConfigMap**: Per file di configurazione e variabili d'ambiente non sensibili.
- **Secret**: Per password, chiavi API e certificati.

## Esercizio consigliato
Prova a eseguire il Deployment di esempio fornito nella cartella:
```bash
kubectl apply -f deployment.yaml
kubectl get pods
```

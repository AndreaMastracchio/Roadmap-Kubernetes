# Modulo 5: 💾 Storage

I Pod sono effimeri: se un Pod muore e viene ricreato, i file salvati localmente vengono persi. Per i database o per dati persistenti serve lo storage persistente.

## 1. PersistentVolumes (PV)
Una risorsa di storage fornita dall'amministratore del cluster o da un cloud provider (es. un disco AWS EBS o un volume NFS).

## 2. PersistentVolumeClaims (PVC)
È la richiesta di storage fatta da uno sviluppatore. "Voglio 10GB di spazio di tipo ReadWriteOnce". Kubernetes troverà un PV adatto e lo legherà al PVC.

## 3. StorageClasses
Permettono il provisioning dinamico: quando crei un PVC, Kubernetes può creare automaticamente il PV per te.

## Esempio YAML
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```
Nel Pod, puoi montare questo PVC:
```yaml
spec:
  containers:
  - name: db
    volumeMounts:
    - mountPath: "/data"
      name: my-storage
  volumes:
  - name: my-storage
    persistentVolumeClaim:
      claimName: my-pvc
```

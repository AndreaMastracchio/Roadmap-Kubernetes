# Modulo 4: 🌐 Networking Avanzato

In Kubernetes, ogni Pod ha il suo indirizzo IP unico. Il networking gestisce come questi Pod comunicano tra loro e con il mondo esterno.

## 1. Ingress
Mentre un `Service` di tipo `LoadBalancer` espone un solo servizio alla volta (spesso con un costo aggiuntivo in cloud), un `Ingress` agisce come un router/reverse proxy intelligente (spesso basato su Nginx o Traefik) che può gestire più servizi basandosi sul percorso URL o sul dominio.

Esempio di Ingress:
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-app-ingress
spec:
  rules:
  - host: myapp.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: hello-service
            port:
              number: 80
```

## 2. Network Policies
Di default, in Kubernetes tutti i Pod possono parlare con tutti gli altri. Le `NetworkPolicies` permettono di limitare il traffico (come un firewall per Pod).
È una best practice di sicurezza fondamentale.

## 3. CNI (Container Network Interface)
Kubernetes non implementa il networking da solo, ma usa plugin come:
- **Calico**: Molto usato per le Network Policies.
- **Flannel**: Semplice e veloce.
- **Cilium**: Basato su eBPF, molto performante.

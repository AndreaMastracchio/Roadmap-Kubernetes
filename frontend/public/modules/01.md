# Modulo 1: 🏗️ Fondamentali dei Container

Prima di poter usare Kubernetes, devi capire cos'è un container e come funziona.

## Perché i Container?
I container permettono di isolare un'applicazione e le sue dipendenze, garantendo che funzioni nello stesso modo in qualsiasi ambiente (locale, staging, produzione).

## Docker
Docker è lo strumento più comune per creare e gestire container.

### Comandi fondamentali da conoscere:
- `docker build`: Per creare un'immagine da un `Dockerfile`.
- `docker run`: Per avviare un container da un'immagine.
- `docker ps`: Per vedere i container in esecuzione.
- `docker images`: Per elencare le immagini scaricate o create.
- `docker exec`: Per entrare in un container attivo.

### Esempio Dockerfile
```dockerfile
FROM golang:1.21-alpine
WORKDIR /app
COPY . .
RUN go build -o main .
CMD ["./main"]
```

## Esercizio consigliato
1. Crea una piccola applicazione "Hello World" in Go.
2. Scrivi un `Dockerfile` per pacchettizzarla.
3. Esegui il container localmente e verifica che risponda correttamente.

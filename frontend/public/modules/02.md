# Modulo 2: 🏛️ Architettura di Kubernetes

Comprendere come Kubernetes gestisce i carichi di lavoro.

## 🧠 Control Plane (Il "Cervello")
Componenti responsabili del mantenimento dello stato desiderato del cluster:
- **kube-apiserver**: Il punto di ingresso per tutte le richieste API (anche `kubectl`).
- **etcd**: Database chiave-valore per lo storage persistente dei dati del cluster.
- **kube-scheduler**: Decide su quale nodo far girare i nuovi Pod.
- **kube-controller-manager**: Gestisce i cicli di controllo (es. riparare un nodo fallito).

## 👷 Worker Nodes (Le "Braccia")
I nodi dove girano effettivamente le applicazioni:
- **kubelet**: L'agente che gira su ogni nodo e si assicura che i container siano avviati.
- **kube-proxy**: Gestisce il networking per i servizi del cluster.
- **Container Runtime**: Il software che esegue i container (es. containerd, Docker).

## Strumenti per iniziare
Per studiare Kubernetes in locale, puoi usare:
- **Minikube**: Un cluster Kubernetes a nodo singolo facile da installare.
- **Kind (Kubernetes in Docker)**: Esegue i nodi Kubernetes come container Docker. Molto veloce.

## Comandi kubectl
Il tool principale per interagire con il cluster:
- `kubectl cluster-info`: Verifica lo stato del cluster.
- `kubectl get nodes`: Elenca i nodi del cluster.
